# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

### Build
```bash
npm run build          # Build dist files + generate ESM modules
npm run build+test     # Build then run full test suite
```

Windows equivalents use `build-windows`, `test-windows`, `build+test-windows`.

### Test
```bash
npm run test           # Full suite: lint + mocha + playwright + lockfile + dts
npm run test:mocha     # Unit tests only (fastest)
npm run test:eslint    # ESLint on all of src/
npm run test:jshint    # JSHint on src/main only
npm run test:playwright # End-to-end tests across all browsers
npm run test:nyc       # Unit tests with coverage report
```

**Run a single test file:**
```bash
npx mocha test/unit/main.js
npx mocha test/unit/submodules/bot-detection.spec.js
```

**Run tests matching a pattern:**
```bash
npx mocha test/unit/main.js --grep "getBrowser"
```

## Architecture

### Source vs. Generated Files
- **`.js` files in `src/`** — canonical source (CommonJS). Edit these.
- **`.mjs` files in `src/`** — auto-generated ESM wrappers. Never edit directly; they are regenerated by `script/build-esm.js` during `npm run build`.
- **`.d.ts` files in `src/`** — hand-maintained TypeScript definitions. Edit when adding or changing the public API.
- **`dist/`** — minified/packed build outputs. Generated by `script/build-dist.sh`.

### Module Layout

| Path | Purpose |
|------|---------|
| `src/main/ua-parser.js` | Core `UAParser` class — all browser, OS, engine, CPU, device detection |
| `src/extensions/` | Optional regex maps for bots, crawlers, CLIs, apps, media players, etc. |
| `src/enums/` | String constants for every browser name, OS name, device type, etc. |
| `src/bot-detection/` | `isBot()` / `isAICrawler()` — standalone bot/crawler helpers |
| `src/browser-detection/` | `isChromeFamily()`, `isElectron()`, `isFromEU()`, `isStandalonePWA()` |
| `src/device-detection/` | `getDeviceVendor()`, `isAppleSilicon()` |
| `src/helpers/` | Deprecated shims that forward to the above submodules |
| `script/cli.js` | CLI entry point (`ua-parser-js` binary) |

### Core Design

**`UAParser` class** (`src/main/ua-parser.js`) wraps a UA string and optional Client Hints headers. Each getter (`getBrowser()`, `getOS()`, `getEngine()`, `getCPU()`, `getDevice()`) returns a result object that exposes:
- Named properties (`name`, `version`, `type`, `vendor`, etc.)
- `.is(value)` — type-check against an enum value
- `.toString()` — formatted string representation

All detection is regex-based. The main file contains large compiled regex maps organized by detection category.

**Submodules** (`bot-detection`, `browser-detection`, `device-detection`, `enums`, `extensions`, `helpers`) are independently importable via package exports for tree-shaking. Each ships as a CJS/ESM pair plus a `.d.ts`.

**Extensions** are additive regex maps passed to the `UAParser` constructor as the second argument. They allow callers to extend detection without modifying the core. `src/extensions/ua-parser-extensions.js` ships a curated set (Bots, Crawlers, CLIs, Apps, etc.).

### Test Structure

Unit tests are data-driven: JSON fixtures in `test/data/ua/<category>/` define `{ desc, ua, expect }` objects, loaded and iterated by Mocha tests in `test/unit/`. Submodule tests live in `test/unit/submodules/`. E2E tests use Playwright (`test/e2e/`). Fuzz tests targeting ReDoS are in `test/fuzz/`.

### Package Exports

```
"."                → src/main/ua-parser.js   (CJS) / .mjs (ESM)
"./bot-detection"  → src/bot-detection/
"./browser-detection" → src/browser-detection/
"./device-detection"  → src/device-detection/
"./enums"          → src/enums/
"./extensions"     → src/extensions/
"./helpers"        → src/helpers/  (deprecated)
```

`"browser"` field points to `dist/ua-parser.pack.js` (bundled UMD for CDN use).
